# 数据源清单与采集注意事项

本文件供 skill 在「步骤 2 采集」时按需查阅。所有数据源均为**公开访问**，不涉及登录。

## 一、平台热榜 / 热点

### 国内
- 抖音热榜：在抖音网页端/搜索页查「抖音热榜」；`web_search "抖音热榜 今日"` 或 `web_fetch` 抖音热点页。
- 巨量算数（抖音官方内容趋势）：`web_search "巨量算数 热点 <关键词>"`，看行业热词、上升趋势。
- 微博热搜、知乎热榜、B站热门：作为跨平台热点交叉验证。
- 聚合工具（可选）：TrendRadar 一类开源热点聚合项目，可一次看多平台榜单。

### 海外（后续平台开启时用）
- YouTube：Trending 页、YouTube 搜索的近期高播放。
- X (Twitter)：Trending topics、相关关键词高互动推文。
- TikTok：Creative Center / Discover 的上升话题与热门音乐。
- Google Trends：`web_fetch` trends.google.com 看关键词上升趋势。

## 二、竞品近期爆款
- 逐个打开 `config.yaml` 里 `competitors` 的主页链接（用 `web_fetch` 或托管浏览器读公开主页）。
- 找近 7-14 天里，数据明显高于该账号平均水平的内容。
- 每条记录：标题/文案、开头前 3 秒讲了什么、内容结构、题材角度、可见的量化信号（赞/评/藏/上榜）。
- 数据抓不准时标注"估算"或"无数据"，不编造。

## 三、行业新闻事件
- `web_search` 组合关键词，覆盖三条产品线：
  - AI / 智能体：新模型、新 agent 产品、行业争议、能力突破。
  - 云托管：算力/托管/部署相关动态、价格战、故障事件。
  - AI 硬件 / 陪伴玩具：新品发布、众筹、评测、社会话题（如"AI 陪伴""能干活的玩具"）。
- 关注「有时效性、能蹭、且能落地到卖点」的事件，避免纯资讯堆砌。

## 采集注意事项
- 优先 `web_search` / `web_fetch`；需要渲染或展开交互时才用托管浏览器，且只访问公开页面。
- 控制频率，避免对同一站点高频抓取。
- 始终保留来源链接与时间，便于人工核验和日报溯源。
- 不确定的数字一律标注来源与"估算/无数据"。

## 部署（在 OpenClaw 上运行）

### 安装 skill
1. 把整个 `trend-topic-radar/` 目录放入 OpenClaw 的 skills 目录（或发布到 ClawHub 后安装）。
2. 复制 `config.example.yaml` 为 `config.yaml`，补全产品线卖点、目标人群、竞品账号。

### 手动运行
在与 OpenClaw 的对话里说：「用选题雷达跑一遍今天的选题」即可触发。

### 配定时任务（每天自动产出）
用 OpenClaw 的调度能力设一个每天早上的任务，示例意图：
```
每天 08:30 执行：运行 trend-topic-radar，产出今日选题日报并发给我审核。
```
产出后 skill 会把日报归档到 `选题库/YYYY-MM-DD.md`，并在对话里提示你挑选。

### 与后续模块的衔接（预留）
本 skill 只到"选题草稿"为止。挑选后的选题，后续可交给：
- `content-publish`（一稿多平台改写 + 定时发布，草稿制人工审核）
- `unified-inbox`（私信/评论/通知聚合）
- `analytics`（数据复盘、爆款规律沉淀）
